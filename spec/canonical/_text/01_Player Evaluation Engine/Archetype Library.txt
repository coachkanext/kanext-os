Archetype Library
1. Purpose
The Archetype Library defines functional player categories used to describe how a player typically impacts the game.
Archetypes translate evaluated attributes into clear basketball roles without assigning value, rank, or usage priority.
Archetypes are descriptive.
They are not evaluative.
2. What an Archetype Is
An archetype represents:
a recognizable on-court role
defined by required skill thresholds
expressed through trait combinations
Archetypes describe how a player tends to contribute, not how good the player is overall.
A player may qualify for:
one archetype
multiple archetypes
no archetype
3. What an Archetype Is Not
An archetype is not:
a position
a lineup instruction
a coaching recommendation
a usage mandate
a projection of future success

Archetypes do not:
determine playing time
dictate offensive or defensive systems
replace coaching judgment
4. Structure of the Archetype Library
Each archetype definition includes:
an identity description
required “must-have” trait thresholds
supporting trait thresholds
explicit disqualifiers
Thresholds are:
fixed
transparent
applied consistently
There are no hidden adjustments.
5. Qualification Rules
Archetype qualification is:
rule-based
deterministic
repeatable
If a player meets the defined criteria, the archetype applies.
If the criteria are not met, the archetype does not apply.
There is no partial credit.
6. Multiple Archetypes
A player may qualify for more than one archetype.
When this occurs:
all qualifying archetypes are recorded
no prioritization is applied at this stage
no conflict resolution occurs within the Archetype Library
Tradeoffs are handled downstream.
7. Independence from Context
Archetypes do not change based on:
team system
opponent
role assignment
playing time
wins or losses
Archetypes reflect player capability, not usage.
8. Downstream Use
Archetypes may be used by:
position-based weighting
team construction analysis
recruiting evaluation
scouting preparation
development explanation
Archetypes may not be modified by downstream processes.
9. Authority and Change Control
The Archetype Library is authoritative for:
archetype definitions
qualification thresholds
disqualifier rules
Any change to:
thresholds
definitions
disqualifiers
requires documentation, versioning, and approval.
