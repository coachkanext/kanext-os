Offensive System Inference Engine (OSIE)
(Governed, Runnable, PrediXt-Compatible)
1. Purpose (Locked)
The Offensive System Inference Engine (OSIE) identifies what offensive system a team actually runs, how fast they play, and how confidently that system can be inferred.
OSIE is descriptive only.
OSIE does not:
modify player ratings
alter archetypes
affect system fit
influence valuation
change any KR
OSIE outputs are labels + confidence, consumed downstream by:
Predixt Simulations simulation
Matchup interaction layers
Calibration & variance logic (DOC 6)
2. Outputs (Locked)
OSIE always outputs the following:
2.1 Primary Offensive System
One of the locked offensive system set (Section 3).
2.2 System Confidence %
A numeric confidence score (0–100%) representing certainty that the Primary System is correct.
This value:
feeds PrediXt variance inflation
governs mixture simulation
does NOT change means
2.3 System Mix (Conditional)
Returned only if dominance criteria are not met.
Includes:
Top 2 (or Top 3, max) systems
Mix Shares (%) that sum to 100%
2.4 Offensive Pace Profile
Outputs:
PACE100 (numeric)
Pace Band: Fast / Neutral / Slow
Used by:
possession estimation
live simulation conditioning (DOC 8)
2.5 Heliocentric Anchor Position (Conditional)
Returned only if Primary System = Heliocentric.
Anchor position ∈:
PG / CG / Wing / Forward / Big

Primary System still returned

9. Dominance Rule (Locked)

10. System Mix Share Computation (NEW, Locked)
When System Mix is returned:
Take top N system scores (N = 2 or 3)
Normalize scores into percentage shares:
Mix Share = System Score / Σ(System Scores)

Shares must sum to 100%
Primary system = highest share
11. System Confidence % Computation (NEW, Locked)
Confidence is derived from:
Dominance margin
Sample size
Data completeness
Governed mapping:
Clear dominance + full data → 85–95%
Moderate dominance → 70–84%
Mix required → 50–69%
Sparse / missing data → ≤ 50%
Confidence never alters system selection, only downstream variance.

OSIE outputs labels + confidence only.


Defensive System Inference Engine
Defensive System Inference Engine
Purpose
The Defensive System Inference Engine (DSIE) identifies what defensive system a team actually runs, how pressure is applied, and how confidently that structure can be inferred.
DSIE is descriptive only.
It labels defensive structure.
It does not evaluate defensive quality.
Role Within Projection
DSIE operates upstream of simulation and downstream of evaluation.
Its outputs are used to:
parameterize defensive behavior in simulation
condition system × system interactions
govern archetype × system interactions
calibrate variance and pressure effects
DSIE outputs are inputs, not conclusions.
Authority and Non-Authority (Locked)
DSIE is authorized to:
classify defensive system identity
return system mixtures when dominance is unclear
classify defensive court depth
compute and report confidence
DSIE is not authorized to:
modify player KRs
modify team KRs
alter archetypes
affect system fit or valuation
influence usage or rotation
change simulation logic
All DSIE outputs are labels + confidence only.
Determinism
DSIE is fully deterministic.
Given the same inputs:
the same system label is returned
the same mix shares are returned
the same confidence values are returned
the same court depth classification is returned
There is no learning, tuning, or adaptation.
System Identity Governance
Defensive system identities:
are drawn from a locked system set
use fixed system IDs
must exactly match IDs used in:
System × System interaction
Archetype × System interaction
Possession Engine logic
No aliases are permitted.
Confidence and Mixture Handling
Confidence:
reflects certainty of inference
does not alter system identity
does not alter means
governs variance inflation downstream
System Mix:
is returned only when dominance criteria are not met
includes a maximum of two systems
always sums to 100%
Primary system remains the highest share.
Defensive Court Depth
Defensive court depth:
is classified independently of system identity
describes pressure location, not scheme
is consumed by possession pressure and pace logic
Court depth never overrides system identity.
Downstream Use
DSIE outputs may be consumed by:
System × System interaction
Archetype × System interaction
Possession Engine
Calibration and variance logic
Live simulation conditioning
DSIE outputs may not be:
interpreted as coaching intent
interpreted as effectiveness
used as evaluation signals
