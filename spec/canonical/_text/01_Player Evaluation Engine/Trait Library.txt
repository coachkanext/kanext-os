Trait Library
1. Purpose
The Trait Library defines the atomic basketball skills used to evaluate individual players.
attributes represent repeatable on-court abilities that can be measured, compared, and scored using consistent rules.
Each trait:
describes one specific skill
is evaluated independently
does not assume role, position, or system
attributes are the building blocks of all player evaluation.
2. What a Trait Is
A trait is:
a single basketball skill
observed during live play
measured using defined inputs
scored using fixed calibration rules
Examples include:
types of shooting
finishing ability
passing actions
defensive behaviors
rebounding actions
physical movement and capacity
A trait is not:
a role
a position
a lineup fit
a coaching preference
a projection

3. Structure of the Trait Library
The Trait Library is organized into skill groups, including:
Shooting
Finishing
Playmaking
Perimeter Defense
Interior Defense
Rebounding
Physical
Each group contains multiple attributes.
Each trait definition includes:
a clear description
required data inputs
event tagging rules
calibration thresholds
a scoring method
4. Trait Evaluation Rules
All trait evaluations are:
deterministic
rule-based
repeatable
Given the same inputs, the same trait score is produced.
Trait scores:
do not change based on team context
do not change based on opponent
do not change based on outcomes
do not change based on role or usage
5. Independence of attributes
Each trait is evaluated independently.
attributes do not:
adjust for other attributes
compensate for weaknesses elsewhere
imply overall player value
Tradeoffs are handled later in the evaluation process.
6. Use of Volume and Efficiency
Where applicable, trait scores may consider:
efficiency
volume
or both
The weighting of efficiency versus volume is:
defined per trait
fixed
disclosed within the trait definition
No hidden weighting is applied.
7. Evidence vs Evaluation
Some data collected during trait evaluation is used as:
evidence, not scoring input
Examples include:
right/left splits
situational tags
secondary action tags
This information supports:
KaNeXT badges
confidence handling
misfit identification
It does not change the trait score.
8. What attributes Do Not Do
Trait scores do not:
rank players
assign roles
predict success
recommend usage
determine playing time
attributes describe what a player can do, not what a player should do.
9. Downstream Use
Trait scores may be used by:
archetype classification
position weighting
KR evaluation
KaNeXT badges
confidence and risk handling
Trait scores may not be modified by downstream processes.
10. Authority
The Trait Library is authoritative for:
defining skill units
evaluating those skills
producing trait-level scores
Changes to trait definitions require:
documentation
versioning
explicit approval

SHOOTING CLUSTER — TRAIT CALIBRATION
Trait 1: 3PT — Spot-Up (Catch-and-Shoot)
Definition:
All stationary catch-and-shoot three-point attempts (no dribble).

Inputs (Primary — locked):
Catch-and-shoot 3P%

Catch-and-shoot 3PA per game

Event Tagging Fields (used to compute inputs):

Defender distance at release

Contest classification (open / guarded / contested)

Shot location (corner / wing / slot)

Time from catch to release

Shooter balance state (set / drifting / off-balance)

Math:
pct_band = Band(C&S 3P%)

vol_band = Band(C&S 3PA/G)

Score = round(0.70 * pct_band + 0.30 * vol_band)
