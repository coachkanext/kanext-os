1. PRODUCT OVERVIEW
================================================================================

KaNeXT OS is a cross-platform mobile operating system for organizations. It
serves five institutional modes — Sports, Enterprise, Church, Education, and
Community — through a single universal shell with mode-specific instantiations.

Every mode shares the same 5-tab navigation, the same UI primitives, and the
same AI layer (Nexus). What changes per mode is the data, the labels, the
domain logic, and the operational vocabulary.

The app is built for one user identity at a time — currently **Sammy Kalejaiye**
(Head Coach / GM) — who can switch between modes to manage different
organizational domains from a single device.

Primary instantiation (Sports Mode):
  - Organization: Florida Memorial University (FMU)
  - Program: FMU Monarchs Men's Basketball
  - Conference: Sun Conference (NAIA)

Secondary instantiations:
  - Enterprise: KaNeXT Inc. (founder OS)
  - Church: International Christian Center Los Angeles (ICCLA)
  - Education: FMU (academic lens)
  - Community: K-1 Speed Racing League

7. NEXUS TAB (AI SURFACE)
================================================================================

Component: NexusScreen (wrapped in NexusProvider when authenticated)
Single conversational canvas — no sub-tabs.

Two states:
  - Locked (unauthenticated): NexusLockedState — "Sign in to unlock Nexus"
  - Unlocked: Full conversational AI surface with GPT backend

Panels and overlays (slide in without leaving screen):
  - ConversationsPanel      — hamburger icon → conversation list
  - ProgramContextDrawer    — context button → program config side drawer
  - RosterOverlay           — board button (1st press) → roster reference
  - RecruitingOverlay       — board button (2nd press) → recruiting board
  - SimulationOverlay       — new sim from conversations
  - VoiceOverlay            — mic button / long-press Nexus tab
  - AvatarDrawer            — avatar press in ConversationsPanel
  - InsertSheet             — "+" button in InputBar
  - NewConversationSheet    — create new conversation

Conversation types: general, player_eval, team_eval, simulation, game_ops,
recruiting, development, film_analysis

Nexus uses program context (system preset, offensive/defensive styles, roster,
recruiting board, cluster weights) to ground all AI responses in the user's
actual program truth.

15. CANONICAL SYSTEMS (SPORTS MODE)
================================================================================

15.1 TEAM KR & CLUSTER SYSTEM (v1 — LOCKED)
─────────────────────────────────────────────
  Team KR = Team Off KR (53%) + Team Def KR (47%), range 0–100

  7 Canonical Clusters (immutable):
    Offense: Shooting, Finishing, Playmaking
    Defense: On-Ball Defense, Team Defense, Rebounding
    Physical: Physical

  Offense clusters: minutes × usage weighted
  Defense/Physical: minutes-weighted only

  Each cluster has subclusters (atomic units, canonical library objects)
  54 total sub-traits across 7 clusters

  Rollup chain:
    Player Subclusters → Team Subclusters → Team Clusters →
    Off KR / Def KR → Team KR

  Team KR is NEVER manually edited — always derived from player truths.

15.2 GAME PLAN (11 Sections)
──────────────────────────────
  S01: Decision Summary
  S02: Opponent Offense
  S03: Opponent Defense
  S04: Shot Profile
  S05: Actions & Triggers

15.3 SIMULATION ENGINE (9 Types)

15.4 RECRUITING PIPELINE (10 Stages)
