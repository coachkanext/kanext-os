System Fit (Offensive)
System Fit (Offensive)
1. Purpose
System Fit (Offensive) defines how player offensive impact expresses itself within a specific offensive system selected in Coach Program Setup.
It answers one question:
Given this offensive system, how effectively can each player’s offensive skill set translate and scale?
This section produces contextual offensive truth.
It does not redefine the player.
2. Relationship to Player Truth
System Fit (Offensive):
consumes finalized Player Intelligence outputs
does not change attributes
does not change raw KRs
does not change archetype assignment
does not change confidence values
Player Truth remains immutable.
System Fit applies interpretive constraints and ceilings only.
3. When This Section Executes
System Fit (Offensive) executes:
after Coach Program Setup
after Coach & Program Context Layer
before Team Evaluation
If no offensive system is declared, this section does not run.
4. What Offensive System Fit Does
System Fit (Offensive) is authorized to:
apply system-specific weighting emphasis
apply offensive fit ceilings
surface offensive leverage and friction
flag offensive system risks
It may:
reduce effective offensive contribution
cap offensive ceiling
explain underutilization or inefficiency
5. What Offensive System Fit Does Not Do
System Fit (Offensive) does not:
increase player skill values
create new offensive abilities
override KaNeXT badges or overrides
compensate for missing skills
guarantee usage or production
System Fit explains compatibility, not capability creation.
6. Determinism
System Fit (Offensive) is:
rule-based
deterministic
repeatable
Given the same player truth and the same offensive system, the same offensive system-fit result is produced.

7. Output
This section produces:
system-adjusted offensive contribution values
offensive fit ceilings
offensive system risk flags
These outputs are consumed by:
Team Evaluation
Team Final Evaluation Output
Roster Sandbox scenarios
They do not exist independently.
8. Governance Rule
System Fit (Offensive):
never alters Player Truth
never persists outside the selected context
is recalculated whenever offensive system selection changes
Changing the offensive system creates a new contextual team truth.

System Fit (Defensive)
System Fit (Defensive)
1. Purpose
System Fit (Defensive) defines how player defensive impact expresses itself within a specific defensive system selected in Coach Program Setup.
It answers one question:
Given this defensive system, how effectively can each player’s defensive skill set translate and hold up?
This section produces contextual defensive truth.
It does not redefine the player.
2. Relationship to Player Truth
System Fit (Defensive):
consumes finalized Player Intelligence outputs
does not change attributes
does not change raw KRs
does not change archetype assignment
does not change confidence values
Player Truth remains immutable.
3. When This Section Executes
System Fit (Defensive) executes:
after Coach Program Setup
after Coach & Program Context Layer
before Team Evaluation
If no defensive system is declared, this section does not run.

4. What Defensive System Fit Does
System Fit (Defensive) is authorized to:
apply system-specific defensive emphasis
apply defensive fit ceilings
surface structural defensive risks
identify scheme-dependent strengths and weaknesses
It may:
reduce effective defensive contribution
cap defensive impact
flag fragility in certain matchups
5. What Defensive System Fit Does Not Do
System Fit (Defensive) does not:
inflate defensive ability
mask defensive weaknesses
override player risks
compensate for physical or positional limitations
System Fit reflects survivability and scalability, not optimism.
6. Determinism
System Fit (Defensive) is:
rule-based
deterministic
repeatable
Given the same player truth and the same defensive system, the same defensive system-fit result is produced.
7. Output
This section produces:
system-adjusted defensive contribution values
defensive fit ceilings
defensive system risk flags
These outputs feed directly into:
Team Evaluation
Team Final Evaluation Output
Roster Sandbox modeling
They are not standalone evaluations.
8. Governance Rule
System Fit (Defensive):
never alters Player Truth
never persists outside the selected context
is recalculated whenever defensive system selection changes
Changing the defensive system creates a new contextual team truth.
