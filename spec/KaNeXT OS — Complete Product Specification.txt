===============================================================================
1. PRODUCT OVERVIEW
===============================================================================

KaNeXT OS is a cross-platform mobile application built with Expo and React
Native. It is a universal operational intelligence platform that serves five
distinct organizational modes from a single codebase:

  1. Sports    — Athletic program management (basketball-first, FMU demo)
  2. Enterprise — Founder/investor operating system (OSK Group, KaNeXT demo)
  3. Church    — Ministry operations (ICCLA demo)
  4. Education — Academic institution management (SDCC demo)
  5. Community — Competition/league management (K-1 League demo)

The app targets iOS (primary), Android, and web via Expo Router file-based
routing. The AI reasoning surface ("Nexus") uses GPT-4o for conversational
intelligence scoped to the active mode and organization context.

User Identity (Demo): Sammy Kalejaiye — Head Coach / GM
Avatar: assets/images/sammy-kalejaiye.jpg

4. MODE SYSTEM
===============================================================================

File: context/app-context.tsx

Mode = 'sports' | 'enterprise' | 'church' | 'education' | 'community'

Switching modes:
  - First launch: ModeGate screen (2-column grid, tap = immediate commit)
  - After: Long-press Organization tab → ModeSwitcherOverlay popup
  - KXTransition micro-flash animation plays on switch

Each mode controls:
  - Home screen dashboard + tab list
  - Organization screen tab list
  - Media sub-labels (Film Room / Ministry Rooms / Classrooms / etc.)
  - Nexus AI system prompt context
  - Data layer (different mock data per mode)

--------------------------------------
File: components/ui/paged-tab-bar.tsx

- Active tab: 2.5px bottom indicator, font-weight 700

- Auto-scrolls to page containing active tab on change
- Includes safe area top padding (insets.top) — ALWAYS first element
- Same Y-level on every screen (uniform positioning)
- OUTSIDE scrollable content area (fixed/sticky)

6.2 Edge-Hold Auto-Advance
----------------------------
File: components/ui/edge-hold-advance.tsx

- 28px invisible Pressable zones on left/right screen edges
- Hold 300ms to begin auto-advancing
- Initial interval: 550ms per step
- Accelerates to 450ms after 3 steps
- Haptic feedback (Light impact) each step
- Clamps to [0, tabCount-1], no wrap-around
- Uses absolutely-positioned Pressable (NOT Gesture.Manual — Expo Go compat)

12. COMPUTATION ENGINES (UTILS)
===============================================================================

12.1 KR Computation (utils/fit-kr.ts)
---------------------------------------
Core Knowledge Rating engine. System-adjusted player ratings.

Key exports:
  computeFitKR(clusters, offStyle, defStyle): number       — 0-100 fit score
  computePositionKR(clusters, position): number            — position-lens KR
  canPlayPosition(clusters, targetPosition): boolean       — flexibility check
  deriveArchetype(clusters, position): Archetype           — archetype derivation
  computeLineupRating(starters, off, def, mins): { offKR, defKR, netKR }
  computeOffDefKR(clusters, off, def): { baseOff, baseDef, fitOff, fitDef }
  getFitReasons(clusters, archetypes, off, def): FitReason[]

Cluster types: shooting, finishing, playmaking, perimeter_defense,
  interior_defense, rebounding, frame
Position weights: PG/CG/W/F/B, derived from Off KR (53%) / Def KR (47%)

12.2 Player Badges (utils/player-badges.ts)
---------------------------------------------
Computes badges from cluster + subcluster ratings.
Badge caps: Max 1 Gold (>=97), 3 Silver (>=94), unlimited Bronze (>=90).
22 badges: 13 offensive + 9 defensive.

12.5 OpenAI Integration (utils/openai.ts)

13. CANONICAL ENGINE SPECIFICATIONS (REFERENCE)
===============================================================================

Located in: spec/canonical/_text/

These are the authoritative, locked specifications for the KaNeXT evaluation
and simulation engines. They are NOT implemented as code in the app —
they define the rules that the AI reasoning system (Nexus) must follow.

13.1 Engine 00 — Coach Master Input
-------------------------------------
Required inputs for program context setup: Program Name, Governing Body,
Division, Offensive/Defensive System. Optional: Conference, Scholarships,
NIL Pool, Operating Budget, Staffing Capacity.

13.2 Engine 01 — Player Evaluation Engine
-------------------------------------------

Archetype Library:
  21 named archetypes (descriptive, not evaluative). Each has must-have
  traits, supporting traits, and auto-disqualifiers.

Badge Cap & Effect Spec:
  College thresholds: Bronze >=90, Silver >=94, Gold >=97
  College effects: Bronze +3, Silver +6, Gold +10
  Global caps: Max 1 Gold, max 3 Silver, Bronze uncapped
  Overall impact cap: College <=+3.5 to Final KR

Player KR Legend:
