1. PRODUCT OVERVIEW
================================================================================

KaNeXT OS is a cross-platform mobile operating system for organizations. It
serves five institutional modes — Sports, Business, Church, Education, and
Competition — through a single universal shell with mode-specific instantiations.

Every mode shares the same 5-tab navigation, the same UI primitives, and the
same AI layer (Nexus). What changes per mode is the data, the labels, the
domain logic, and the operational vocabulary.

The app is built for one user identity at a time — currently **Sammy Kalejaiye**
(Head Coach / GM) — who can switch between modes to manage different
organizational domains from a single device.

Primary instantiation (Sports Mode):
  - Organization: Florida Memorial University (FMU)
  - Program: FMU Monarchs Men's Basketball
  - Conference: Sun Conference (NAIA)

Secondary instantiations:
  - Business: KaNeXT Inc. (founder OS)
  - Church: International Christian Center Los Angeles (ICCLA)
  - Education: FMU + Bethune-Cookman University (BCU) (academic lens)
  - Competition: K-1 Speed Racing League

3.2 RBAC LENS SYSTEMS
──────────────────────
Each mode uses a 5-tier Role Lens system for fine-grained visibility gating.

  Sports Role Lens (utils/sports-rbac.ts):
    R1 — AD / HC / GM          Full access (all tabs, all drill-downs)
    R2 — Player                Self-only (cards view, no recruiting/game-plan)
    R3 — Asst Coach / RC       Limited eval, most tabs
    R4 — Scout / Analyst       Limited eval, no game-plan/sim/development
    R5 — Fan                   Public only, cards-only roster, no recruiting

  Church Role Lens (utils/church-rbac.ts):
    C1 — Senior Pastor         Full access
    C2 — Elder / Board         Full access
    C3 — Staff / Deacon        Most tabs, admin restricted
    C4 — Member                Limited tabs (overview + member-facing)
    C5 — Visitor / Public      Locked or curated view

  Education Role Lens (utils/education-rbac.ts):
    E1 — President/Chancellor  Full access
    E2 — Provost / Dean        Full access
    E3 — Faculty / Staff       Most tabs, admin restricted
    E4 — Student               Limited tabs (overview + student-facing)
    E5 — Public                Locked or curated view

  Business Role Lens (utils/business-rbac.ts):
    B1  — Founder / CEO        Full access
    B2a — Investor (Retail)    Limited disclosure
    B2b — Investor (Strategic) Enhanced disclosure
    B3  — Public               Minimal access
    B4  — Subscriber           Curated access
    B5  — Prospective Acquirer Due diligence views

  Competition Role Lens (utils/competition-rbac.ts):
    Per-role gating for league_admin, team_owner, driver.

7. NEXUS TAB (AI SURFACE)
================================================================================

Component: NexusScreen (wrapped in NexusProvider when authenticated)
Single conversational canvas — no sub-tabs.

Two states:
  - Locked (unauthenticated): NexusLockedState — "Sign in to unlock Nexus"
  - Unlocked: Full conversational AI surface with GPT-4o backend

Panels and overlays (slide in without leaving screen):
  - ConversationsPanel      — hamburger icon → conversation list
  - ProgramContextDrawer    — context button → program config side drawer
  - RosterOverlay           — board button (1st press) → roster reference
  - RecruitingOverlay       — board button (2nd press) → recruiting board
  - SimulationOverlay       — new sim from conversations
  - VoiceOverlay            — mic button / long-press Nexus tab
  - AvatarDrawer            — avatar press in ConversationsPanel
  - InsertSheet             — "+" button in InputBar
  - NewConversationSheet    — create new conversation

Conversation types: general, player_eval, team_eval, simulation, game_ops,
recruiting, development, film_analysis

Player Query System (utils/nexus-player-query.ts):
  - Auto-detects player-related queries via keyword+regex analysis
  - Extracts filters from natural language (level, position, KR range, etc.)
  - Searches real national pool (9K+ players) via nationalPool.search()
  - Injects matched player data as [PLAYER DATA] block into GPT context
  - GPT responds with real statistical data, not hallucinated numbers

V2 Governed Actions (utils/nexus-actions.ts):
  - Action classification: "add player to board", "send message", etc.
  - Pending action confirmation flow before execution
  - Intent classification via voice-classifier.ts

Pool Awareness Prompt (always injected in sports mode):
  - Describes 9K+ player database, KR methodology, 7-cluster system
  - Teaches GPT the [LINK:player:ID:Name] tappable format

                         F (Forward), B (Big)

  7 Canonical Clusters:
    Offense: Shooting, Finishing, Playmaking
    Defense: Perimeter Defense, Interior Defense, Rebounding
    Physical: Frame

  Position Cluster Weights (College):
    PG: OKR 55%, DKR 40%, TKR 5%
    CG: OKR 60%, DKR 35%, TKR 5%
    W:  OKR 50%, DKR 40%, TKR 10%
    F:  OKR 45%, DKR 40%, TKR 15%
    B:  OKR 35%, DKR 45%, TKR 20%

  KR Color Bands (10 universal bands — utils/kr-display.ts):
    97–100: Gold #FFD700 (Elite/Transcendent)

  Level-Aware Tier Labels (KR_LEGEND): 13 competitive levels with

18. KR COMPUTATION ENGINE
================================================================================

Location: services/player-pool/engine/ (8 Python modules)
Run: cd services/player-pool/engine && python3 compute_engine.py
Version: v1.0-boxscore

7-Step Pipeline:
  1. Season stats aggregation — per-player, per-season
  2. BPR (Basketball Performance Rating) — zero-centered, per-game z-score
  3. PGIS (Per-Game Impact Score) — system-aware cluster multipliers
  4. TGIS (Team Game Impact Score) — team-level game impact
  5. Player KR — KLVN normalization + position weights → archetype
  6. Team KR — participation-weighted (53% OFF / 47% DEF locked split)
  7. OSIE/DSIE — system inference (11 OFF + 9 DEF systems)

Post-pipeline: Scholarship & NIL Allocation Engine (scholarship_nil.py)

Engine Results (latest run):
  3,646 Player KRs computed
  32,362 BPR scores
  32,362 PGIS scores
  1,578 TGIS scores
  388 Team KRs
  685 OSIE/DSIE entries
  3,524 Scholarship/NIL allocations

Modules:
  compute_engine.py    — Main orchestrator
  bpr.py               — Basketball Performance Rating
  clusters.py          — 7-cluster score computation (0-100 each)
  player_kr.py         — Position-weighted KR + archetype derivation
  team_kr.py           — Team KR aggregation
  osie_dsie.py         — System Inference Engine
  klvn.py              — Level normalization parameters
  impact_scores.py     — PGIS, TGIS, Impact Modifiers
  scholarship_nil.py   — Scholarship & NIL Allocation Engine

NOT YET BUILT:
  - Pro KR (separate position weights, archetypes, badge thresholds)
  - badges.py (full badge computation in engine — currently frontend-only)
  - kr_legend.py (tier definitions for engine)
