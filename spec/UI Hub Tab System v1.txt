KaNeXT OS — Hub Tab System Specification
=========================================
Version: 1.0 (Locked)
Date: 2026-02-16
Supersedes: All prior "More" overflow menu behavior


1. OVERVIEW
-----------
Every primary surface (Home, Media, Messages, Organization) uses a shared
PagedTabBar component for sub-navigation. Tabs are horizontally scrollable
in pages of 4. There is NO "More" overflow menu anywhere in the system.

All tabs that were previously hidden behind "More" menus are now promoted
to real, visible tabs in the scrollable tab row.


2. PAGED TAB BAR (Universal Component)
---------------------------------------
File: components/ui/paged-tab-bar.tsx

Behavior:
- Renders tabs in a horizontal ScrollView with page snapping
- Each tab is fixed width = screenWidth / 4 (exactly 4 per page)
- Active tab has a 2.5px bottom indicator line
- When active tab changes, the tab row auto-scrolls to the page
  containing that tab
- Includes safe area top padding (insets.top) so it sits below
  the status bar on all screens uniformly

Visual Spec:
- Font: 14px, weight 500 (inactive), weight 700 (active)
- Letter spacing: 0.2
- Active color: colors.text (white in dark mode)
- Inactive color: colors.textTertiary (#424242)
- Bottom border: StyleSheet.hairlineWidth, colors.divider
- Tab vertical padding: 12px

Position Rule (LOCKED):
- PagedTabBar is ALWAYS the first rendered element on every screen
- It sits at the exact same Y-level on every screen (Home, Media,
  Messages, Organization) regardless of mode
- Nothing renders above it except the system status bar
- It is OUTSIDE the scrollable content area (fixed/sticky)


3. EDGE-HOLD AUTO-ADVANCE
--------------------------
File: components/ui/edge-hold-advance.tsx

Power-user gesture: hold the left or right screen edge to step through
tabs automatically.

Behavior:
- 28px invisible touch zones on left and right screen edges
- Hold for 300ms to begin auto-advancing
- Initial interval: 550ms per step
- Accelerates to 450ms after 3 steps
- Each step triggers haptic feedback (Light impact)
- Bounds: clamps to [0, tabCount-1], no wrap-around
- Releasing the edge stops advancing immediately

Implementation:
- Uses absolutely-positioned Pressable zones (NOT Gesture.Manual,
  which conflicts with PagerView in Expo Go)
- onPressIn starts hold timer, onPressOut clears all timers


4. TAB PLACEHOLDER PAGE
------------------------
File: components/ui/tab-placeholder-page.tsx

Tabs without real content render a centered placeholder card:
- Title: tab name
- Subtitle: "This section is not built yet."
- Themed background and text colors

6. ORGANIZATION SCREEN LAYOUT
------------------------------
The Organization screen renders NO header (no title, no cycle label,
no org-switcher button). The PagedTabBar is the first element, matching
every other screen. Organization switching is handled via the
OrgSwitcherSheet (triggered by mode sub-components as needed).

8. REMOVED SYSTEMS
-------------------
The following are permanently removed:

- utils/global-more.ts — Global "More" menu controller (deleted)
- "More" overflow menu on all screens (removed)
- Double-tap Organization tab to open More (removed)
- registerMoreHandlers / unregisterMoreHandlers pattern (removed)
- Organization screen title header (removed)
- CompanySwitcher above Enterprise org tabs (removed)
- Roster tier badge ("Regional Power") (removed)

The Organization tab bottom-bar button retains:
- Single tap: navigate to Organization screen
- Long-press: open Mode Switcher
