1. PRODUCT OVERVIEW
================================================================================

KaNeXT OS is a cross-platform mobile operating system for organizations. It
serves five institutional modes — Sports, Enterprise, Church, Education, and
Community — through a single universal shell with mode-specific instantiations.

Every mode shares the same 5-tab navigation, the same UI primitives, and the
same AI layer (Nexus). What changes per mode is the data, the labels, the
domain logic, and the operational vocabulary.

The app is built for one user identity at a time — currently **Sammy Kalejaiye**
(Head Coach / GM) — who can switch between modes to manage different
organizational domains from a single device.

Primary instantiation (Sports Mode):
  - Organization: Florida Memorial University (FMU)
  - Program: FMU Monarchs Men's Basketball
  - Conference: Sun Conference (NAIA)

Secondary instantiations:
  - Enterprise: KaNeXT Inc. (founder OS)
  - Church: International Christian Center Los Angeles (ICCLA)
  - Education: FMU + Bethune-Cookman University (BCU) (academic lens)
  - Community: K-1 Speed Racing League

3.2 RBAC LENS SYSTEMS (V2 Organization Tabs)
──────────────────────────────────────────────
V2 organization tabs use a 5-tier Role Lens system for fine-grained visibility
gating within each tab. Each lens maps to a tier number (1 = highest authority,
5 = most restricted).

  Church Role Lens (utils/church-rbac.ts):
    C1 — Senior Pastor        Full 10-tab access
    C2 — Elder / Board        Full 10-tab access
    C3 — Staff / Deacon       Most tabs, some admin areas restricted
    C4 — Member               Limited tabs (overview + member-facing)
    C5 — Visitor / Public     Locked or curated view only

  Education Role Lens (utils/education-rbac.ts):
    E1 — President/Chancellor Full 10-tab access
    E2 — Provost / Dean       Full 10-tab access
    E3 — Faculty / Staff      Most tabs, admin areas restricted
    E4 — Student              Limited tabs (overview + student-facing)
    E5 — Public               Locked or curated view only

  Business Role Lens (utils/business-rbac.ts):
    B1  — Founder / CEO       Full access
    B2a — Investor (Retail)   Limited disclosure views
    B2b — Investor (Strategic/Board)  Enhanced disclosure views
    B3  — Public              Minimal access
    B4  — Subscriber          Curated access
    B5  — Prospective Acquirer  Due diligence views

  Helper functions per mode: isDeanLevel(), isFacultyLevel(), isEnrolled(),
  isPresident(), isPastorLevel(), isStaffLevel(), isMember(), etc.
  Each v2 component receives a `role` prop and gates sub-tabs + content
  visibility accordingly.

7. NEXUS TAB (AI SURFACE)
================================================================================

Component: NexusScreen (wrapped in NexusProvider when authenticated)
Single conversational canvas — no sub-tabs.

Two states:
  - Locked (unauthenticated): NexusLockedState — "Sign in to unlock Nexus"
  - Unlocked: Full conversational AI surface with GPT backend

Panels and overlays (slide in without leaving screen):
  - ConversationsPanel      — hamburger icon → conversation list
  - ProgramContextDrawer    — context button → program config side drawer
  - RosterOverlay           — board button (1st press) → roster reference
  - RecruitingOverlay       — board button (2nd press) → recruiting board
  - SimulationOverlay       — new sim from conversations
  - VoiceOverlay            — mic button / long-press Nexus tab
  - AvatarDrawer            — avatar press in ConversationsPanel
  - InsertSheet             — "+" button in InputBar
  - NewConversationSheet    — create new conversation

Conversation types: general, player_eval, team_eval, simulation, game_ops,
recruiting, development, film_analysis

Nexus uses program context (system preset, offensive/defensive styles, roster,
recruiting board, cluster weights) to ground all AI responses in the user's
actual program truth.

10. V2 ORGANIZATION ARCHITECTURE
================================================================================

V2 organization tabs follow a standardized architectural pattern across all
modes. Each v2 tab is a self-contained component with its own mock data file.

10.1 STRUCTURAL PATTERN
────────────────────────
Every v2 component follows this internal structure:

  1. JSDoc header (component name, sub-tab list, RBAC rules)
  2. Imports (React, RN, ThemedText, IconSymbol, BottomSheet, theme, RBAC utils)
  3. SUB_TABS constant (array of { id, label } — always 10 entries)
  4. Props interface ({ colors, accentColor, role })
  5. Helper functions (formatDate, formatCurrency, daysUntil)
  6. Reusable sub-components (EmptyState, StatusBadge, ProgressBar, SubTabBar)
  7. 10 sub-tab content functions (each a React component)
  8. Detail bottom sheets (2-4 per component, all using useModal)
  9. Main export function with RBAC gating + state + renderContent switch
 10. StyleSheet.create block

10.2 SUB-TAB BAR
─────────────────
Each v2 component renders a horizontal ScrollView tab bar at the top with
10 sub-tabs. The active sub-tab determines which content function renders.
This is NOT a PagerView — it's a simple state-driven switch with haptic
feedback on tab press.

10.3 RBAC GATING PATTERN
──────────────────────────
  - Highest-restricted roles (C5/E5/B3) → Locked state (lock icon + message)
  - Mid-restricted roles (C4/E4) → Limited sub-tab access (2-4 tabs)
  - Staff roles (C3/E3) → Most sub-tabs, admin areas restricted
  - Leadership roles (C1-C2/E1-E2/B1) → Full 10-tab access

  The visibleSubTabs useMemo filters the SUB_TABS array based on role.
  Content functions also have inline RBAC guards for sensitive data (e.g.,
  financial values hidden from student roles).

10.4 DETAIL BOTTOM SHEETS
──────────────────────────
Each v2 component includes 2-4 detail bottom sheets using the BottomSheet
wrapper with useModal prop (portal-based for deeply nested rendering).
Sheets follow the standard 50%/100% snap point pattern.

10.5 MOCK DATA FILE PATTERN
─────────────────────────────
Each v2 component has a dedicated mock data file that follows this structure:

  1. Type aliases (union types for all domain enums)
  2. Interfaces (exported, one per data entity)
  3. Label/color/icon constant maps (Record<Type, string>)
  4. Seeded data arrays (realistic, domain-appropriate)
  5. Factory function (e.g., getEduComplianceData())

All types, interfaces, and label/color maps are exported for use by
the component. The factory function returns a flat object with all
data collections.

10.6 INTERACTION PATTERNS
──────────────────────────
  - All Pressable elements trigger expo-haptics (ImpactFeedbackStyle.Light
    for navigation, Medium for actions)
  - FlatList for list-heavy tabs, ScrollView for non-list tabs
  - useCallback/useMemo for render optimization
  - contentContainerStyle includes paddingBottom: 120 for safe scrolling

12. UI SYSTEM & PRIMITIVES
================================================================================

12.1 CORE PRIMITIVES (components/ui/)
──────────────────────────────────────
  PagedTabBar        — scrollable paged horizontal tab bar (4 tabs per page,
                       screenWidth/4 per tab). Haptic feedback. Used by all
                       swipeable hub surfaces.
  EdgeHoldAdvance    — left/right edge-hold auto-advance gesture (28px zone,
                       300ms hold threshold, 550→450ms ramp after 3 steps)
  BottomSheet        — universal @gorhom/bottom-sheet wrapper. Two snap points
                       (50%/100%). All sheets open at 50% first. Drag-to-dismiss
                       from anywhere. Backdrop tap dismisses.
  IconSymbol         — cross-platform icon: SF Symbols (iOS) / MaterialIcons
  Collapsible        — animated expand/collapse with chevron
  AskNexusCTA        — gold-bordered contextual deep-link to Nexus
  VideoHeroCard      — 16:9 autoplay/loop/muted hero video
  EmptyState         — standardized empty content placeholder
  ErrorState         — standardized error display
  LoadingState       — standardized loading indicator
  TabPlaceholderPage — generic "Coming Soon" placeholder (no longer used)

12.2 THEMED COMPONENTS
──────────────────────
  ThemedText         — auto light/dark text primitive
  ThemedView         — auto light/dark view primitive
  GlobalHeader       — persistent header shown across all screens
  HapticTab          — tab button with haptic feedback

12.3 BOTTOM SHEET RULES (LOCKED)
────────────────────────────────
  - Two snap points ONLY: 50% (medium) and 100% (full)
  - All sheets open at 50% first
  - Drag-to-dismiss from ANYWHERE inside the sheet
  - Backdrop tap dismisses (40% opacity dim)
  - Spring animation (no linear, no abrupt)
  - PROHIBITED: fixed-height modals, close-button-only, center modals,
    >2 snap points, custom Animated.View/PanResponder implementations

12.4 THEMING

    Defense: On-Ball Defense, Team Defense, Rebounding
