1. PRODUCT OVERVIEW
================================================================================

KaNeXT OS is a cross-platform mobile operating system for organizations. It
serves five institutional modes — Sports, Business, Church, Education, and
Competition — through a single universal shell with mode-specific instantiations.

Every mode shares the same 5-tab navigation, the same UI primitives, and the
same AI layer (Nexus). What changes per mode is the data, the labels, the
domain logic, the role hierarchy, and the operational vocabulary.

The app is built for one user identity at a time — currently **Sammy Kalejaiye**
(System Owner) — who can switch between modes to manage different organizational
domains from a single device.

Primary instantiation (Sports Mode):
  - Organization: Lincoln University
  - Program: Men's Basketball
  - Location: Oakland, CA

Secondary instantiations (live):
  - Business: KaNeXT (founder OS) — Atlanta, GA
  - Church: ICC — Los Angeles, CA

Coming Soon (configured with data, orgs, and RBAC — not yet live in UI):
  - Education: Howard University — Washington, DC
  - Competition: Adidas 3SSB — Rock Hill, SC

a RoleLens union type and visibility matrices that map (tab, role) → visibility.

4.6 SYSTEM OWNER (X0)
───────────────────────

The SYSTEM_OWNER is a KaNeXT-internal-only role — mode-agnostic, org-agnostic,
single instance (founder account only). Never visible to or assignable by any org.

Resolves to the highest-privilege lens in each mode:
  Sports → R0, Business → B0, Church → C0, Education → E0, Competition → CO0

Capabilities: allModes, crossOrg, lensSwitch, debugOverride, globalAudit

4.7 UNIFIED RBAC DISPATCHER
─────────────────────────────

resolveRoleLens(membershipId, mode):
  1. isSystemOwner()? → X0 lens (short-circuit)
  2. mode-specific role getter → RoleLens
  3. Returns AnyRoleLens { mode, lens }

AnyRoleLens = discriminated union across all 5 mode lens types.

8. NEXUS TAB (AI SURFACE)
================================================================================

Component: NexusScreen (wrapped in NexusProvider when authenticated)
Single conversational canvas — no sub-tabs.

8.1 STATES
───────────
  - Locked (unauthenticated): Dimmed "KX" logo + "Sign in to unlock Nexus"
  - Onboarding (new user): Auto-create conversation, fetch GPT welcome, mark complete
  - Unlocked: Full conversational AI surface with GPT-4o backend

8.2 LEFT PANEL — ConversationsPanel
─────────────────────────────────────
  - Shows all conversations (pinning, renaming, archiving, deletion)
  - "New Chat" + "New Sim" buttons
  - Avatar press → opens drawer

8.3 MAIN CONTENT AREA
───────────────────────
  - Landing: NexusLanding component (when no active conversation)
  - Chat Thread: ChatThread component (when conversation active)
  - Game Ops Sub-header: "Carroll vs {opponent}" when game-ops active
  - Input Bar: InputBar with mic, attach (+), send

8.4 RIGHT PANELS & OVERLAYS
─────────────────────────────
  - ProgramContextDrawer   — context button → program config side drawer
  - RosterOverlay          — board button (1st press) → roster reference
  - RecruitingOverlay      — board button (2nd press) → recruiting board
  - SimulationOverlay      — new sim from conversations
  - VoiceOverlay           — mic button / long-press Nexus tab
  - InsertSheet            — "+" button → action menu

8.5 CONVERSATION TYPES
────────────────────────
general, player_eval, team_eval, simulation, game_ops,
recruiting, development, film_analysis

8.6 PLAYER QUERY SYSTEM (utils/nexus-player-query.ts)
───────────────────────────────────────────────────────
  - Auto-detects player-related queries via 3 keyword sets + 16 regex patterns
  - Keyword sets: player (20+ terms), position (9), level (10)
  - Extracts filters from natural language (level, position, KR range, etc.)
  - Searches real national pool (9K+ players) via nationalPool.search()
  - Injects matched player data as [PLAYER DATA] block into GPT context
  - GPT responds with real statistical data, not hallucinated numbers

8.7 GOVERNED ACTIONS (utils/nexus-actions.ts)
──────────────────────────────────────────────
  - Intent classifier: pattern-matching natural language → ActionIntent (no GPT)
  - Types: create_task, create_request, post_room, approve, deny, escalate,
    flag_player, change_pipeline_stage, and more
  - Pending action confirmation flow before execution

8.8 GAME OPS FLOW
───────────────────
  - Triggered via global handler from game-detail screen
  - Creates game-ops conversation with config (gameId, opponent, step, periodFormat, starters)
  - User types naturally, GPT parses and asks clarifying questions

8.9 SPEECH RECOGNITION
───────────────────────
  - Mic press: startListening(), records to voiceState
  - Audio level visualization on VoiceOverlay
  - Stop recording: stopListening() on overlay close

17.1 KR (KANEXT RATING) SYSTEM
────────────────────────────────
  KR = composite player quality rating, range 0–100.
  Computed from 7 canonical clusters via position-weighted KLVN SkillKR pipeline.

  5 Canonical Positions: PG (Point Guard), CG (Combo Guard), W (Wing),
                         F (Forward), B (Big)

  7 Canonical Clusters:
    Offense: Shooting, Finishing, Playmaking
    Defense: On-Ball Defense, Team Defense, Rebounding
    Physical: Physical (Frame)

  KLVN SkillKR Pipeline v1.2 (per sub-trait):
    1. Bayesian shrinkage: p_adj = (N*p + α*μ) / (N + α)
    2. Expected efficiency: E(v) = μ (MVP constant from calibration)
    3. Variance normalization: Z = (p_adj - μ) / (σ0/√N)
    4. Sigmoid mapping: SkillKR = 100 / (1 + exp(-1.2 * Z)) → 0-100

  20 sub-traits across 7 clusters, each with per-level empirical parameters.
  Calibration: calibrate.py → klvn_params.json (empirical μ/σ0 per trait × level).

  Overall KR: 0.53 * OffKR + 0.47 * DefKR (canonical split)
  kr_status: FINAL (confidence ≥ 60%) or PROVISIONAL

  KR Color Bands (10 universal bands — utils/kr-display.ts):
    97–100: #1D9BF0  Elite/Transcendent
    94–96:  #A1A1AA  Franchise Anchor
    90–93:  #1D9BF0  High-Impact
    86–89:  #1D9BF0  Solid Starter
    82–85:  #1D9BF0  Trusted Rotation
    78–81:  #22C55E  Reliable Bench
    74–77:  #F59E0B  Situational
    70–73:  #F59E0B  Limited
    66–69:  #EF4444  Fringe/Project
    0–65:   #A1A1AA  Below Viability

  Level-Aware Tier Labels (13 competitive levels):
    ncaa_d1_high_major, ncaa_d1_mid_major, ncaa_d1_low_major,
    ncaa_d2, ncaa_d3, naia, njcaa_d1, njcaa_d2, njcaa_d3,
    cccaa, uscaa, nccaa_d1, nccaa_d2
    Each level has custom tier breakpoints (higher levels = higher thresholds).
